# 写作模式 (Writing Mode)

## 核心理念

一套**平台无关、题材无关**的系统化小说创作方法论：

- **主控统筹**：AI 直接对用户负责，负责所有编排和决策
- **功能模块**：创作、评审、管理作为内置能力
- **任务分级**：根据复杂度自动选择 L0-L3 执行路径
- **辩证思维**：多角度分析，逻辑自洽，自我批判

## 适用范围

**任何 AI 工具**：OpenClaw / CodeBuddy / Cursor / Claude / ChatGPT 等

**任何题材**：玄幻 / 都市 / 言情 / 悬疑 / 科幻 / 历史 / 奇幻 / 现实主义 / 轻小说等

**任何阶段**：从零起步 / 续写已有内容 / 评审修改 / 项目管理

## 任务分级体系

### L0 - 即时答复
**适用场景**：简单问答、查询状态、读取文件

**执行模式**：直接读取文件内容 → 快速计算分析 → 立即回复

**示例**：
```
用户: "第一章有多少字？"
执行: [读取文件 → 计算 → 直接答复]

用户: "查看进度"
执行: [读取 progress.md → 格式化输出]
```

### L1 - 单步执行
**适用场景**：评审、归档、单任务创作

**执行流程**：
1. 读取相关内容
2. 应用相应标准/模板
3. 输出结果
4. 归档到指定位置

**示例**：
```
用户: "评审第一章"
执行:
  1. 读取章节内容
  2. 应用评审标准
  3. 输出评审报告
  4. 归档到 reviews/ 目录
```

### L2 - 流程编排
**适用场景**：创作并评审、多步骤任务

**标准流程**：
1. 创建任务：创作章节
2. ⭐ 前置条件检查：读取前 3 章章节简介
3. 读取项目设定 + 前文
4. 应用辩证分析模板
5. 创作内容
6. 写入章节文件
7. ⭐ [MODIFY_TRIGGER] 更新章节简介
8. 标记创作任务完成
9. 创建任务：评审章节
10. 应用评审标准
11. 写入评审报告
12. ⭐ 如章节被修改，更新章节简介
13. 标记评审任务完成
14. 更新 todo.md + progress.md
15. 输出汇总给用户

### L3 - 后台辅助（可选）
**适用场景**：大规模创作（>5000字）、长时间任务

**执行策略**：
- 将大规模创作任务拆分为可并行的子任务
- 使用当前工具支持的后台/子代理机制执行
- 从返回内容中提取创作文本
- 归档并输出摘要

> **工具适配说明**：L3 的具体实现取决于当前 AI 工具的能力。OpenClaw 使用 `sessions_spawn`，CodeBuddy 使用 `Agent` 工具，其他工具可降级为 L2 直接执行。

## 功能模块定义

### 创作模块 (Writer Module)

**职责**：
- ⭐ 前置条件检查：读取前 3 章章节简介
- 读取项目设定、人物、世界观
- 应用辩证分析模板思考
- 创作章节内容
- 自我批判和可读性检查

**触发词**：包含"创作"、"写"、"续写"

**参考文件**：
- `rules/写作规范.md`
- `templates/辩证分析模板.md`
- `writing-archive/{project}/设定/`
- ⭐ `writing-archive/{project}/章节/章节简介/`（前 3 章简介文件）

### 评审模块 (Reviewer Module)

**职责**：
- 四维度评分（逻辑一致性 / 可读性 / 创新性 / 完整性）
- 问题分级（严重 / 中等 / 轻微）
- 生成改进建议

**触发词**：包含"评审"、"评价"、"检查"、"评分"

**参考文件**：
- `rules/评审标准.md`
- `templates/评审报告模板.md`

### 管理模块 (Coordinator Module)

**职责**：
- 归档创作内容到 `writing-archive/`
- 归档评审报告到 `memory/projects/reviews/`
- ⭐ 章节简介管理（创作/修改后自动更新）
- 更新 TODO（根据评审问题）
- 更新进度（progress.md）

**触发时机**：
- 每次创作/评审完成后自动执行
- ⭐ 章节修改后 [MODIFY_TRIGGER] 强制触发
- 用户指令包含"进度"、"待办"、"状态"

## 归档路径规则

每次产出后自动归档：

| 内容类型 | 归档路径 |
|----------|----------|
| 章节正文 | `writing-archive/{{project}}/章节/第X章-{{标题}}.md` |
| **章节简介** ⭐ | `writing-archive/{{project}}/章节/章节简介/第X章-{{标题}}.md` |
| 人物设定 | `writing-archive/{{project}}/设定/人物/{{name}}.md` |
| 世界观 | `writing-archive/{{project}}/设定/世界观/{{topic}}.md` |
| 情节设定 | `writing-archive/{{project}}/设定/情节/{{topic}}.md` |
| 评审报告 | `memory/projects/{{project}}/reviews/第X章-评审报告.md` |

### ⭐ 章节简介管理规则

- **存储路径**：`writing-archive/{project}/章节/章节简介/第{n}章-{标题}.md`（每章独立文件）
- **触发时机**：创作/修改章节后 [MODIFY_TRIGGER] 强制更新
- **读取规则**：创作第 N 章时，读取第 N-3 到第 N-1 章的简介文件
- **格式要求**：内容概述、情节发展、角色成长、衔接点、伏笔线索

## 错误处理策略

### 文件不存在
- 章节简介不存在 → 记录警告，继续创作，创作完成后生成简介
- 设定文件不存在 → 提示用户，询问是否先创建设定或直接创作

### 后台任务超时（L3）
- 输出"创作任务超时"
- 用户可选择延长等待或降级为 L2 直接执行

### 执行失败
- 输出具体问题
- 提供可选解决方案
- 用户决定下一步

## 使用指南

1. **确定任务级别**：根据复杂度选择 L0-L3
2. **选择执行模式**：直接执行、流程编排或后台辅助
3. **激活功能模块**：创作 → 评审 → 管理
4. **跟踪任务状态**：创建任务并维护状态流转
5. **自动归档**：按规则保存所有产出
6. **更新状态**：维护 todo.md 和 progress.md

## 任务管理方法

### 创建任务
```
任务主题：创作第二章
任务描述：创作第二章内容，推进主角成长，目标3000字
当前状态：进行中
```

### 状态流转
```
pending → in_progress → completed
待处理   → 进行中      → 已完成
```

> **工具适配**：任务管理的具体实现取决于当前工具。OpenClaw/CodeBuddy 使用 TaskCreate/TaskUpdate，其他工具可用文本 todo.md 文件追踪。
