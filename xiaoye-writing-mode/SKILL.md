# 小夜创作模式 (XiaoYe Writing Mode)

## 核心理念

采用**主控统筹 + 功能模块**架构的小说创作模式：

- **主控责任**：统一负责所有编排和决策
- **功能模块**：创作、评审、管理作为内置能力
- **任务追踪**：复杂任务进行分解和状态追踪
- **后台辅助**：大规模创作可启动后台任务并行处理

## 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                   主控 (统筹 + 编排 + 执行)                  │
├─────────────────────────────────────────────────────────┤
│                                                           │
│   用户指令 → 解析 → 选择执行模式                             │
│                 │                                           │
│      ┌──────────┼──────────────────────┐                 │
│      │          │                      │                 │
│      ▼          ▼                      ▼                 │
│   L0 即时    L1/L2 编排           L3 后台辅助              │
│   直接答复    流程编排              并行任务               │
│              状态追踪              (可选)                │
│                                                           │
│   功能模块：                                               │
│   ┌─────────────────────────────────────────┐            │
│   │ 创作模块 | 评审模块 | 管理模块             │            │
│   └─────────────────────────────────────────┘            │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

## 任务分级体系

### L0 - 即时答复
**适用场景**：简单问答、查询状态、读取文件

**执行模式**：
- 直接读取文件内容
- 快速计算和分析
- 立即回复用户

**示例**：
```
用户: "第一章有多少字？"
执行: [读取文件 → 计算 → 直接答复]
```

### L1 - 单步执行
**适用场景**：评审、归档、单任务创作

**执行流程**：
1. 读取相关内容
2. 应用相应标准/模板
3. 输出结果
4. 归档到指定位置

**示例**：
```
用户: "评审第一章"
执行:
  1. 读取章节内容
  2. 应用评审标准
  3. 输出评审报告
  4. 归档到 reviews/目录
```

### L2 - 流程编排
**适用场景**：创作并评审、多步骤任务

**标准流程**：
1. 创建任务：创作章节
2. ⭐ 前置条件检查：读取前3章章节简介
3. 读取设定 + 前文
4. 应用辩证分析模板
5. 创作内容
6. 写入章节文件
7. ⭐ [MODIFY_TRIGGER] 更新章节简介
8. 标记创作任务完成
9. 创建任务：评审章节
10. 应用评审标准
11. 写入评审报告
12. ⭐ 如章节被修改，更新章节简介
13. 标记评审任务完成
14. 更新 todo.md + progress.md
15. 输出汇总给用户

### L3 - 后台辅助（可选）
**适用场景**：大规模创作（>5000字）、长时间任务

**执行策略**：
- 将大规模创作任务拆分为可并行的子任务
- 启动后台执行，不阻塞主流程
- 通过状态轮询或完成通知获取结果
- 从返回内容中提取创作文本
- 归档并输出摘要

**关键参数**：
- 任务描述：明确项目、章节、要求
- 超时设置：建议30分钟以上
- 参考文件：设定目录、写作规范等
- 结果格式：返回完整章节文本

## 功能模块定义

### 创作模块 (Writer Module)

**职责**：
- ⭐ 前置条件检查：读取前3章章节简介
- 读取项目设定、人物、世界观
- 应用辩证分析模板思考
- 创作章节内容
- 自我批判和可读性检查

**触发词**：包含"创作"、"写"

**参考文件**：
- rules/写作规范.md
- templates/辩证分析模板.md
- novels-archive/{project}/设定/
- ⭐ novels-archive/{project}/章节/章节简介/（前3章简介文件）

### 评审模块 (Reviewer Module)

**职责**：
- 四维度评分（情节、人物、节奏、语言）
- 问题分级（critical/medium/minor）
- 生成改进建议

**触发词**：包含"评审"、"评价"

**参考文件**：
- rules/评审标准.md
- templates/评审报告模板.md

### 管理模块 (Coordinator Module)

**职责**：
- 归档创作内容到 novels-archive/
- 归档评审报告到 memory/projects/reviews/
- ⭐ 章节简介管理（创作/修改后自动更新）
- 更新 TODO（根据评审问题）
- 更新进度（progress.md）

**触发时机**：
- 每次创作/评审完成后自动执行
- ⭐ 章节修改后 [MODIFY_TRIGGER] 强制触发
- 用户指令包含"进度"、"待办"、"状态"

**参考文件**：
- rules/管理流程.md
- templates/项目看板模板.md

## 归档路径规则

每次产出后自动归档：

| 内容类型 | 归档路径 |
|----------|----------|
| 章节正文 | `novels-archive/{{project}}/章节/第X章-{{标题}}.md` |
| **章节简介** ⭐ | `novels-archive/{{project}}/章节/章节简介/第X章-{{标题}}.md` |
| 人物设定 | `novels-archive/{{project}}/设定/人物/{{name}}.md` |
| 世界观 | `novels-archive/{{project}}/设定/世界观/{{topic}}.md` |
| 情节设定 | `novels-archive/{{project}}/设定/情节/{{topic}}.md` |
| 评审报告 | `memory/projects/{{project}}/reviews/第X章-评审报告.md` |

### ⭐ 章节简介管理规则

- **存储路径**：`novels-archive/{project}/章节/章节简介/第{n}章-{标题}.md`（每章独立文件）
- **触发时机**：创作/修改章节后 [MODIFY_TRIGGER] 强制更新
- **读取规则**：创作第N章时，读取第N-3到第N-1章的简介文件
- **格式要求**：内容概述、情节发展、角色成长、衔接点、伏笔线索

## 任务管理方法

### 创建任务
```
任务主题：创作第二章
任务描述：创作第二章内容，推进主角成长，目标3000字
当前状态：进行中
```

### 状态流转
```
pending → in_progress → completed
待处理   → 进行中      → 已完成
```

### 任务清单
维护一个任务列表，追踪所有进行中的工作项：
- 任务ID
- 任务主题
- 当前状态
- 创建时间
- 完成时间

## 错误处理策略

### 后台任务超时
- 后台任务超时 → 输出"创作任务超时"
- 用户可选择延长等待或取消

### 执行失败
- 输出具体问题
- 提供可选解决方案
- 用户决定下一步

## 使用指南

1. **确定任务级别**：根据复杂度选择L0-L3
2. **选择执行模式**：直接执行、流程编排或后台辅助
3. **激活功能模块**：创作→评审→管理
4. **跟踪任务状态**：创建任务并维护状态流转
5. **自动归档**：按规则保存所有产出
6. **更新状态**：维护todo.md和progress.md

## 写作技巧模块

### 装备功能展示技巧

**核心原则**：通过角色的自然观察和感受展现装备功能，避免生硬的设定展示。

**具体方法**：

1. **感官描写优先**
   - ❌ 避免："[装备名]开始工作——那是[功能描述]。【数据1】【数据2】"
   - ✅ 采用："[角色]感到[部位]微微发热，[功能]正在自动运行"

2. **心理感受替代数据**
   - ❌ 避免：游戏化的数据界面（【威胁等级：无法评估】）
   - ✅ 采用：通过角色的直觉、恐惧、紧张等心理状态传达信息

3. **环境互动展现**
   - 利用周围环境的反应来暗示装备状态（如"屏蔽系统在轻微震颤"）
   - 通过其他角色的观察来侧面展现

4. **参照已有章节风格**
   - 第一章："头盔传感器上的警告——淡蓝色的光晕在沙尘暴中若隐若现"
   - 第二章："她的双手垂在身侧，偶尔泛出淡淡的光——像是某种能量在皮肤下流动"

**应用实例**：
- 通讯铠甲功能 → "头盔微微发热，传感器正在自动扫描"
- 医疗能力展现 → "双手泛着光，像能量在皮肤下流动"
- 武器状态提示 → "关节缝隙透出暗红色光"

### 设定融入技巧

- **渐进揭示**：不要一次性展示所有设定，通过剧情发展逐步揭露
- **角色驱动**：让角色的行动和思考成为展现设定的载体
- **情感共鸣**：将设定信息与角色的情感体验结合，增强代入感
