# 后台任务指南

## 概述

XiaoYe 是单 Agent 主控架构，但支持在需要时使用后台辅助任务处理大规模创作或耗时操作。

**重要**：后台任务是**可选辅助**，不是核心架构。大多数任务由 XiaoYe 直接执行。

---

## 何时使用后台任务

| 场景 | 是否使用后台任务 |
|------|------------------|
| 评审一章（≤3000字） | ❌ 直接执行 |
| 创作一章（≤5000字） | ❌ 直接执行 |
| 创作长章节（>5000字） | ✅ 建议使用 |
| 多章节并行创作 | ✅ 可考虑使用 |
| 大规模设定整理 | ✅ 可考虑使用 |
| 查看进度/待办 | ❌ 直接执行 |

## 方案 A: sessions_spawn（OpenClaw 原生）

适用于 OpenClaw Gateway 环境的子代理调用。

### API 参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `task` | string | ✅ | 子代理执行的任务描述 |
| `model` | string | ❌ | 覆盖模型（默认继承调用者） |
| `thinking` | string | ❌ | 思考级别覆盖 |
| `runTimeoutSeconds` | number | ❌ | 超时秒数（0=无超时） |
| `thread` | boolean | ❌ | 绑定到聊天线程 |
| `mode` | string | ❌ | `"run"`（一次性）或 `"session"`（持久） |
| `cleanup` | string | ❌ | `"keep"` 或 `"delete"` |
| `sandbox` | string | ❌ | `"inherit"` 或 `"require"` |

### 调用示例

```
sessions_spawn(
  task="创作{项目名}第三章：完整的战斗场景，5000字以上，快节奏，细节丰富。",
  model="gpt-4o-mini",
  runTimeoutSeconds=1800,
  mode="run"
)
```

### 返回值

```json
{
  "status": "accepted",
  "runId": "run-abc123",
  "childSessionKey": "agent:xiaoye:subagent:uuid-here"
}
```

### 结果获取

子代理完成时发送**公告**到请求者：

```
Source: subagent
Child Session Key: agent:xiaoye:subagent:uuid
Status: success|error|timeout|unknown
Result: <子代理最后一条可见的 assistant 文本>

Runtime: 5m12s
Tokens: input=12000 output=3500 total=15500
Cost: $0.42
```

**重要**：
- 公告内容是**文本**，不是 JSON
- 主 Agent 需从文本中提取创作内容
- `Status` 来自运行时，不是子代理自述

### 管理命令

```
/subagents list              // 列出活跃子代理
/subagents info <id|#>       // 查看运行详情
/subagents log <id|#>        // 查看日志
/subagents kill <id|#|all>   // 终止子代理
```

## 方案 B: Agent 工具（CodeBuddy）

适用于 CodeBuddy Code CLI 环境。

### 调用格式

```
Agent(
  name="writer-bg",              // 唯一标识名
  description="创作战斗章节",    // 简短描述（3-5词）
  prompt="...",                  // 完整任务指令
  subagent_type="general-purpose",  // 固定类型
  run_in_background=true         // 后台执行
)
```

### Prompt 构造要点

```markdown
# 创作任务

## 项目
{项目名}

## 要求
- 章节：第三章
- 内容：完整的战斗场景
- 字数：5000字以上
- 风格：快节奏、细节丰富

## 参考
请读取以下文件：
- {workspace}/novels-archive/{项目名}/设定/
- {workspace}/rules/写作规范.md
- {workspace}/templates/辩证分析模板.md

## 输出要求
创作完成后，直接返回完整的章节内容文本。
不要返回 JSON 格式，返回纯文本即可。
```

### 结果获取

```
TaskOutput(
  task_id="<Agent返回的task_id>",
  block=true,
  timeout=600000
)
```

### 返回值说明

- Background agent 返回**文本消息**，不是 JSON
- 主 Agent 需自行解析文本内容

## 执行流程示例

### 大规模创作流程

```
用户：创作一个完整的战斗场景章节，5000字以上

XiaoYe 执行：
1. [评估] 任务规模 → 需要后台辅助
2. [TaskCreate] subject="创作战斗章节(5000字)"
3. [后台调用] sessions_spawn 或 Agent 工具
4. [等待] 获取完成公告或 TaskOutput
5. [解析] 从文本中提取创作内容
6. [Write] 归档到 novels-archive/
7. [TaskUpdate] status="completed"
8. [Edit] 更新 progress.md
9. [输出] 创作摘要给用户
```

## 错误处理

### 超时

```
Status: timeout 或 TaskOutput timeout
  → 输出"创作任务超时"
  → 建议用户：等待延长 / 取消 / 缩短字数要求
```

### 执行失败

```
Status: error 或 Agent 执行异常
  → 输出具体错误信息
  → 建议：降级为直接执行 / 缩小任务范围
```

### 结果解析失败

```
无法从返回文本提取创作内容
  → 输出原始返回内容
  → 请用户确认是否使用
```

## 重要事项

### OpenClaw 规范

基于官方文档 (`/tools/subagents.md`)：

1. **非阻塞**：`sessions_spawn` 立即返回，不等待完成
2. **文本返回**：结果是文本公告，不是 JSON
3. **无自动触发**：没有自动链式调用，主 Agent 必须显式编排
4. **完成信号**：子代理的最后一条可见 assistant 消息作为正式结果
5. **禁止污染**：`*_COMPLETE` 之后不得输出 `NO_REPLY`、`ANNOUNCE_SKIP` 等

### CodeBuddy 规范

1. **非阻塞**：`run_in_background=true` 时立即返回
2. **文本返回**：返回文本消息，不是 JSON
3. **显式获取**：使用 `TaskOutput` 获取结果

## 与旧设计的差异

| 旧设计（错误） | 新设计（正确） |
|----------------|----------------|
| `Agent(name="writer", subagent_type="general-purpose")` 作为独立 Subagent | 后台任务是**可选辅助**，不是核心架构 |
| 子 Agent 返回 JSON + suggested_next | 返回**文本**，主 Agent 解析 |
| 存在 auto_triggers 自动触发链 | **不存在自动触发**，需显式编排 |
| 3 个独立 Subagent | 单 Agent + 功能模块 + 可选后台辅助 |
