# 后台任务指南

## 概述

写作模式是单 Agent 主控架构，但支持在需要时使用后台辅助任务处理大规模创作或耗时操作。

**重要**：后台任务是**可选辅助**，不是核心架构。大多数任务由 AI 直接执行。

---

## 何时使用后台任务

| 场景 | 是否使用后台任务 |
|------|------------------|
| 评审一章（≤3000字） | ❌ 直接执行 |
| 创作一章（≤5000字） | ❌ 直接执行 |
| 创作长章节（>5000字） | ✅ 建议使用（如工具支持） |
| 多章节并行创作 | ✅ 可考虑使用 |
| 大规模设定整理 | ✅ 可考虑使用 |
| 查看进度/待办 | ❌ 直接执行 |

---

## 工具适配方案

不同 AI 工具有不同的后台任务实现方式：

### OpenClaw

使用 `sessions_spawn` 启动子代理：

```
sessions_spawn(
  task="创作{项目名}第三章：完整的战斗场景，5000字以上...",
  runTimeoutSeconds=1800,
  mode="run"
)
```

**返回**：
```json
{
  "status": "accepted",
  "runId": "run-abc123",
  "childSessionKey": "agent:subagent:uuid"
}
```

子代理完成时发送**公告**到请求者，主 AI 从公告文本中提取结果。

**管理命令**：
```
/subagents list          // 列出活跃子代理
/subagents info <id>     // 查看运行详情
/subagents log <id>      // 查看日志
```

---

### CodeBuddy

使用 `Agent` 工具启动后台任务：

```
Agent(
  name="writer-bg",
  description="创作战斗章节",
  prompt="...",
  subagent_type="general-purpose",
  run_in_background=true
)
```

使用 `TaskOutput` 获取结果：
```
TaskOutput(
  task_id="<返回的task_id>",
  block=true,
  timeout=600000
)
```

---

### 其他工具 / 降级方案

如果当前工具不支持后台任务，降级为直接执行：

```
策略：
1. 将大章节拆分为多个小段（每段2000-3000字）
2. 逐段创作，每段完成后归档
3. 最后合并为完整章节
4. 更新章节简介
```

## 通用执行流程

无论使用哪种工具，后台任务的逻辑流程相同：

```
用户：创作一个完整的战斗场景章节，5000字以上

AI 执行：
1. [评估] 任务规模 → 需要后台辅助
2. [创建任务] 记录"创作战斗章节(5000字)"
3. [后台调用] 使用当前工具支持的后台机制
4. [等待] 获取完成结果
5. [解析] 从返回内容中提取创作文本
6. [写入] 归档到 writing-archive/
7. [更新] 更新 progress.md + 章节简介
8. [输出] 创作摘要给用户
```

## 后台任务 Prompt 构造要点

无论使用哪种工具，后台任务的 prompt 应包含：

```markdown
# 创作任务

## 项目
{项目名}

## 要求
- 章节：第三章
- 内容：完整的战斗场景
- 字数：5000字以上
- 风格：快节奏、细节丰富

## 参考
请读取以下文件：
- {workspace}/writing-archive/{项目名}/设定/
- {workspace}/rules/写作规范.md
- {workspace}/templates/辩证分析模板.md

## 输出要求
创作完成后，直接返回完整的章节内容文本。
```

## 错误处理

### 超时

```
后台任务超时：
  → 输出"创作任务超时"
  → 建议用户：等待延长 / 取消 / 降级为直接执行 / 缩短字数要求
```

### 执行失败

```
后台任务失败：
  → 输出具体错误信息
  → 建议：降级为直接执行 / 缩小任务范围
```

### 结果解析失败

```
无法从返回内容提取创作文本：
  → 输出原始返回内容
  → 请用户确认是否使用
```

---

## 重要原则

1. **非阻塞**：后台任务启动后立即返回，不等待完成
2. **文本返回**：结果是文本内容，不是 JSON
3. **无自动触发**：没有自动链式调用，主 AI 必须显式编排
4. **降级优先**：工具不支持时，优先降级为直接执行，不要报错
