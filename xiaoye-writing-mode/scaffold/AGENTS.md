# Writing Mode - Agent 配置文档

## 核心理念

Writing Mode 是平台无关的小说创作辅助系统，采用**单 Agent 主控 + 功能模块**架构：

- **主控责任**：AI 直接对用户负责，负责所有编排和决策
- **功能模块**：创作、评审、管理作为内置能力，而非独立 Agent
- **任务追踪**：复杂任务进行分解和状态追踪
- **后台辅助**：大规模创作可使用当前工具支持的后台机制辅助

---

## 任务分级

### L0 - 即时答复

适用：简单问答、查询状态、读取文件

```
用户: "第一章有多少字？"
AI: [读取文件 → 计算 → 直接答复]
```

### L1 - 单步执行

适用：评审、归档、单任务创作

```
用户: "评审第一章"
AI:
  1. 读取章节内容
  2. 应用评审标准（rules/评审标准.md）
  3. 输出评审报告
  4. 归档到 memory/projects/{project}/reviews/
```

### L2 - 流程编排

适用：创作并评审、多步骤任务

```
用户: "创作并评审第三章"
AI:
  1. 创建任务：创作第三章
  2. ⭐ 读取前3章章节简介
  3. 读取设定 + 前文
  4. 应用辩证分析模板
  5. 创作内容
  6. 写入章节文件
  7. ⭐ [MODIFY_TRIGGER] 更新章节简介
  8. 标记创作任务完成
  9. 创建任务：评审第三章
  10. 应用评审标准
  11. 写入评审报告
  12. 标记评审任务完成
  13. 更新 todo.md + progress.md
  14. 输出汇总给用户
```

### L3 - 后台辅助（可选）

适用：大规模创作（>5000字）、长时间任务

```
用户: "创作一个完整的战斗场景章节，5000字以上"
AI:
  1. 创建任务：创作战斗章节(5000字)
  2. 使用当前工具支持的后台/子代理机制启动任务
  3. 等待完成并获取结果
  4. 从返回内容中提取创作文本
  5. 归档到 writing-archive/
  6. 更新 progress.md
  7. 输出摘要给用户
```

**工具适配说明**：

| 工具 | L3 实现方式 |
|------|------------|
| OpenClaw | `sessions_spawn` |
| CodeBuddy | `Agent` 工具 + `TaskOutput` |
| 其他工具 | 降级为 L2 直接执行 |

## 功能模块定义

### 创作模块 (Writer Module)

**定义文件**: `modules/writer-module.md`

**职责**:
- ⭐ 前置条件检查：读取前3章章节简介
- 读取项目设定、人物、世界观
- 应用辩证分析模板思考
- 创作章节内容
- 自我批判和可读性检查

**触发**: 用户指令包含"创作"、"写"、"续写"

**参考文件**:
- `rules/写作规范.md`
- `templates/辩证分析模板.md`
- `writing-archive/{project}/设定/`
- ⭐ `writing-archive/{project}/章节/章节简介/`（前3章简介文件）

---

### 评审模块 (Reviewer Module)

**定义文件**: `modules/reviewer-module.md`

**职责**:
- 四维度评分（逻辑一致性 / 可读性 / 创新性 / 完整性）
- 问题分级（严重 / 中等 / 轻微）
- 生成改进建议

**触发**: 用户指令包含"评审"、"评价"、"检查"、"评分"

**参考文件**:
- `rules/评审标准.md`
- `templates/评审报告模板.md`

---

### 管理模块 (Coordinator Module)

**定义文件**: `modules/coordinator-module.md`

**职责**:
- 归档创作内容到 `writing-archive/`
- 归档评审报告到 `memory/projects/reviews/`
- ⭐ 章节简介管理（创作/修改后自动更新）
- 更新 TODO（根据评审问题）
- 更新进度（progress.md）

**触发**:
- 每次创作/评审完成后自动执行
- ⭐ 章节修改后 [MODIFY_TRIGGER] 强制触发
- 用户指令包含"进度"、"待办"、"状态"

**参考文件**:
- `rules/管理流程.md`
- `templates/项目看板模板.md`

## 归档路径规则

每次产出后由管理模块自动归档：

| 内容类型 | 归档路径 |
|----------|----------|
| 章节正文 | `writing-archive/{{project}}/章节/第X章-{{标题}}.md` |
| **章节简介** ⭐ | `writing-archive/{{project}}/章节/章节简介/第X章-{{标题}}.md` |
| 人物设定 | `writing-archive/{{project}}/设定/人物/{{name}}.md` |
| 世界观 | `writing-archive/{{project}}/设定/世界观/{{topic}}.md` |
| 情节设定 | `writing-archive/{{project}}/设定/情节/{{topic}}.md` |
| 评审报告 | `memory/projects/{{project}}/reviews/第X章-评审报告.md` |

### ⭐ 章节简介管理规则

- **存储路径**：`writing-archive/{project}/章节/章节简介/第{n}章-{标题}.md`（每章独立文件）
- **触发时机**：创作/修改章节后 [MODIFY_TRIGGER] 强制更新
- **读取规则**：创作第N章时，读取第N-3到第N-1章的简介文件
- **格式要求**：内容概述、情节发展、角色成长、衔接点、伏笔线索

## 任务管理

AI 使用任务追踪机制管理复杂任务。具体实现取决于当前工具：

| 工具 | 任务追踪方式 |
|------|------------|
| OpenClaw | `TaskCreate` / `TaskUpdate` |
| CodeBuddy | `TaskCreate` / `TaskUpdate` |
| 其他工具 | 文本 `todo.md` 文件 |

**通用任务格式**：
```
任务主题：创作第二章
任务描述：创作第二章内容，推进主角成长，目标3000字
当前状态：进行中 → 已完成
```

## 快捷指令

| 指令 | 分级 | 执行内容 |
|------|------|----------|
| `写 [章节]` / `创作 [章节]` | L1/L3 | 创作章节 |
| `续写` | L1 | 从上次结尾继续 |
| `评审 [章节]` | L1 | 评审并生成报告 |
| `创作并评审` | L2 | 完整创作+评审+归档+更新TODO |
| `进度` | L0 | 查看 progress.md |
| `待办` | L0 | 查看 todo.md |
| `状态` | L0 | 项目整体状态 |
| `设定 [类型]` | L0 | 查看/创建设定 |
| `人物 [名字]` | L0 | 查看人物卡 |


## 错误处理

### 文件不存在
```
章节简介不存在 → 记录警告，继续创作，完成后生成简介
设定文件不存在 → 提示用户，询问是否先创建设定
```

### 后台任务超时（L3）
```
超时 → 输出"创作任务超时"
     → 用户可选择：延长等待 / 取消 / 降级为L2直接执行
```

### 执行失败
```
任务执行遇到障碍:
  → 输出具体问题
  → 提供可选解决方案
  → 用户决定下一步
```
