# Coordinator Module - 管理模块

## 模块定义

管理模块是 XiaoYe 的文档归档和项目追踪能力，负责维护项目状态、TODO 管理和进度追踪。**这不是独立 Agent，而是 XiaoYe 主 Agent 的内置能力。**

**重要**：每次创作或评审产出后，管理模块自动执行归档和更新。

---

## 核心职责

### 1. 文档归档

- 创作内容 → 归档到 `novels-archive/`
- 评审报告 → 归档到 `memory/projects/reviews/`

### 2. 章节简介管理 ⭐ 新增

- 创作新章节后 → 自动生成对应章节简介
- 修改章节内容后 → **[MODIFY_TRIGGER]** 强制更新对应章节简介
- 评审后修改章节 → **[MODIFY_TRIGGER]** 强制更新对应章节简介
- 章节简介存储路径：`novels-archive/{project}/章节/章节简介.md`
- 简介内容包含：内容概述、情节发展、角色成长、衔接点、伏笔线索

### 3. TODO 管理

- 根据评审问题自动生成修改任务
- 维护任务优先级
- 标记任务完成

### 4. 进度追踪

- 更新章节状态
- 统计字数
- 记录质量趋势

---

## 触发条件

### 自动触发

- **创作完成后**：自动归档创作内容 + **更新章节简介**
- **评审完成后**：自动归档评审报告 + 更新 TODO + 更新进度
- **章节修改后**：**[MODIFY_TRIGGER]** 强制触发 → 更新章节简介

### 手动触发

- 用户指令包含："进度"、"待办"、"状态"、"管理"

## 归档路径规则

| 内容类型 | 归档路径 | 示例 |
|----------|----------|------|
| 章节正文 | `novels-archive/{project}/章节/第{n}章-{标题}.md` | `第3章-血战.md` |
| **章节简介** ⭐ | `novels-archive/{project}/章节/章节简介/第{n}章-{标题}.md` | 每章独立文件 |
| 人物设定 | `novels-archive/{project}/设定/人物/{name}.md` | `主角名.md` |
| 世界观 | `novels-archive/{project}/设定/世界观/{topic}.md` | `设定名.md` |
| 情节设定 | `novels-archive/{project}/设定/情节/{topic}.md` | `主线大纲.md` |
| 评审报告 | `memory/projects/{project}/reviews/第{n}章-评审报告.md` | `第3章-评审报告.md` |

### 章节简介格式

每章简介为独立文件，存储在 `novels-archive/{project}/章节/章节简介/` 目录下。

文件命名：`第{n}章-{标题}.md`

每章简介包含以下结构：

```markdown
# 第{n}章：{标题}

## 内容概述
[100-200字概述本章核心事件]

## 情节发展
- [关键事件和转折点]

## 角色成长
[主要角色的状态变化]

## 衔接点
- [承上启下的关键信息]

## 伏笔线索
- [遗留的伏笔和待解决的线索]

## 最后更新
YYYY-MM-DD
```

## 任务类型

### 归档创作内容

```
触发：Writer Module 完成

执行：
1. 确定归档路径（根据内容类型）
2. Write 内容到目标路径
3. Edit 更新项目索引
4. Edit 更新 progress.md
5. ⭐ [MODIFY_TRIGGER] 更新章节简介（生成或更新对应章节的简介条目）

输出：
- 归档路径确认
- 进度更新摘要
- 章节简介更新确认
```

### 归档评审报告

```
触发：Reviewer Module 完成

执行：
1. Write 评审报告到 reviews/
2. 解析问题级别
3. Edit todo.md 添加任务
4. Edit progress.md 更新分数和趋势
5. ⭐ 如章节被修改，[MODIFY_TRIGGER] 更新章节简介

输出：
- 归档路径确认
- 新增 TODO 列表
- 进度更新摘要
- 章节简介更新确认（如适用）
```

### 更新章节简介 ⭐ 新增

```
触发：章节内容创作/修改后 [MODIFY_TRIGGER]

执行：
1. Read 当前章节内容
2. 提取：内容概述、情节发展、角色成长、衔接点、伏笔线索
3. Read 章节简介.md（如存在）
4. Edit 更新对应章节的简介条目（如不存在则创建文件）
5. 确认更新完成

输出：
- 章节简介更新确认
- 更新内容摘要
```

### 查询项目状态

```
触发：用户指令"进度"、"状态"

执行：
1. Read progress.md
2. Read todo.md
3. Read 项目索引

输出：
- 章节完成情况
- 总字数统计
- 平均评审分
- 质量 trend
- 当前 TODO 清单
```

## 与其他模块的协作

```
创作流程：
[Writer Module] 创作 → [Coordinator Module] 归档 + ⭐更新章节简介

评审流程：
[Reviewer Module] 评审 → [Coordinator Module] 归档 + 更新TODO
    └─ 如章节被修改 → ⭐[MODIFY_TRIGGER] 更新章节简介

修改流程：
[Writer Module] 修改章节 → ⭐[MODIFY_TRIGGER] 更新章节简介

完整流程：
[Writer] → [Coordinator归档+更新简介] → [Reviewer] → [Coordinator归档+更新TODO]
    └─ 如修改 → [Coordinator更新简介]
```
