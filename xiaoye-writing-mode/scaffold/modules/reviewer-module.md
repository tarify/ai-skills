# Reviewer Module - 评审模块

## 模块定义

评审模块是写作模式的质量评审能力，负责对创作内容进行系统评审，提供量化评分和改进建议。**这不是独立 Agent，而是主控 AI 的内置能力。**

---

## 触发条件

- 用户指令包含："评审"、"评估"、"检查"、"评分"
- "创作并评审"流程中的评审环节

---

## 执行流程

```
1. 读取待评审内容
2. 读取评审标准（rules/评审标准.md）
3. 读取项目设定作为基准
4. 四维度逐项评分
5. 识别问题并分级
6. 生成改进建议
7. 写入评审报告到 memory/projects/reviews/
8. 更新 todo.md 和 progress.md
```

---

## 评审标准（四维度）

详细标准见 `rules/评审标准.md`，以下为快速参考：

### 一、逻辑一致性（权重 40%，共 40 分）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 因果链条 | 10分 | 事件发展是否符合逻辑 |
| 人物动机 | 10分 | 行为是否有合理动机 |
| 世界观遵守 | 10分 | 是否违反已建立规则 |
| 角色认知边界 | 10分 | 角色是否说出不该知道的信息 |

### 二、可读性评估（权重 30%，共 30 分）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 语言流畅度 | 5分 | 句式是否自然 |
| 设定融入叙事 | 10分 | 设定是否自然融入，无百科式讲解 |
| 对话合理性 | 10分 | 对话是否符合角色认知水平 |
| 情感传达 | 5分 | 情绪是否真挚 |

### 三、创新性（权重 20%，共 20 分）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 情节设计 | 10分 | 是否有新意 |
| 人物塑造 | 10分 | 是否立体饱满 |

### 四、完整性（权重 10%，共 10 分）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 内容完整 | 5分 | 结构是否完整 |
| 钩子设置 | 5分 | 悬念是否得当 |

## 题材适配评审

不同题材在评审时有不同侧重：

| 题材 | 重点关注 |
|------|---------|
| 玄幻 / 奇幻 | 力量体系逻辑、世界观遵守、设定融入 |
| 都市 / 现实 | 对话合理性、生活细节真实感、情感传达 |
| 言情 | 情感节奏、人物动机、情感传达 |
| 悬疑 / 推理 | 因果链条、线索布局、角色认知边界 |
| 科幻 | 世界观遵守、设定融入叙事、科技感一致性 |
| 历史 | 世界观遵守（时代背景）、对话合理性（时代感） |

---

## 问题分级

| 级别 | 标记 | 说明 | 后续动作 |
|------|------|------|----------|
| 严重 | 🔴 | 影响逻辑或世界观一致性 | 生成 🔴 高优先级 TODO |
| 中等 | 🟡 | 影响阅读体验 | 生成 🟡 中优先级 TODO |
| 轻微 | 🟢 | 可优化但不影响核心 | 生成 🟢 低优先级 TODO |

## 任务类型

### 完整评审（full）

```
输入：
- 章节内容
- 项目设定
- 历史评审（如有）

执行：
- 四维度完整评分
- 详细问题分析
- 改进建议

输出：
- 完整评审报告
- 归档路径：memory/projects/{project}/reviews/第{n}章-评审报告.md
```

### 快速评审（quick）

```
输入：
- 章节内容

执行：
- 核心维度评分（逻辑、可读性、完整性）
- 关键问题识别（最多3个）
- 核心建议（最多3个）

输出：
- 简化评审结果
```

---

## 完成后动作

评审完成后，AI 自动执行：

1. 写入评审报告到 `memory/projects/{project}/reviews/`
2. 根据问题级别，更新 `todo.md` 添加修改任务：
   ```
   - 🔴 修改第{n}章：{问题描述}
   - 🟡 优化第{n}章：{问题描述}
   - 🟢 润色：{问题描述}
   ```
3. 更新 `progress.md` 评审分数和趋势
4. ⭐ **如章节被修改，[MODIFY_TRIGGER] 触发管理模块更新章节简介**

## 示例执行

```
用户：评审第二章

AI 执行：
[读取] writing-archive/{项目名}/章节/第二章-{标题}.md
[读取] rules/评审标准.md
[读取] writing-archive/{项目名}/设定/

[评分]
- 逻辑一致性：32/40（因果链条9，人物动机8，世界观9，认知边界6）
- 可读性：25/30（语言4，设定融入8，对话8，情感5）
- 创新性：17/20（情节9，人物8）
- 完整性：9/10（内容5，钩子4）
- 总分：83/100，⭐4.2

[问题识别]
- 🟡 第3段：战斗描写节奏稍快，可增加过渡
- 🟡 第5段：角色说出了不该知道的信息（认知边界越界）
- 🟢 第7段：对话略显书面化，可口语化

[改进建议]
- 优先级1：修复角色认知边界越界问题
- 优先级2：战斗场景增加感官细节描写
- 优先级3：对话根据人物性格调整语气

[写入] memory/projects/{项目名}/reviews/第2章-评审报告.md

[更新] memory/projects/{项目名}/todo.md
添加：
- 🟡 修改第2章：角色认知边界越界
- 🟡 优化第2章：战斗节奏过渡

[更新] memory/projects/{项目名}/progress.md
更新：最新评审分=4.2，质量趋势=稳定

[输出] 评审完成：总分83分⭐4.2，发现2个中等问题，待办已更新
```

## 与创作模块的协作

在"创作并评审"流程中：

```
[Writer Module] 完成 → 归档 + ⭐更新章节简介
    ↓
[Reviewer Module] 评审 → 归档报告 → 更新TODO
    ↓
    └─ 如章节被修改 → ⭐[MODIFY_TRIGGER] 更新章节简介
    ↓
[输出] 创作摘要 + 评审摘要 + TODO更新
```
