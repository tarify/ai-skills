# Reviewer Module - 评审模块

## 模块定义

评审模块是 XiaoYe 的质量评审能力，负责对创作内容进行系统评审，提供量化评分和改进建议。**这不是独立 Agent，而是 XiaoYe 主 Agent 的内置能力。**

---

## 触发条件

- 用户指令包含："评审"、"评估"、"检查"、"评分"
- "创作并评审"流程中的评审环节

---

## 执行流程

```
1. Read 待评审内容
2. Read 评审标准（rules/评审标准.md）
3. Read 项目设定作为基准
4. 四维度逐项评分
5. 识别问题并分级
6. 生成改进建议
7. Write 评审报告到 memory/projects/reviews/
8. 更新 todo.md 和 progress.md
```

---

## 评审标准（四维度）

### 一、逻辑一致性（权重 30%）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 因果链条 | 10分 | 事件发展是否符合逻辑 |
| 人物动机 | 10分 | 行为是否有合理动机 |
| 世界观遵守 | 10分 | 是否违反已建立规则 |

### 二、可读性评估（权重 30%）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 语言流畅度 | 10分 | 句式是否自然 |
| 信息密度 | 10分 | 重点是否突出 |
| 情感传达 | 10分 | 情绪是否真挚 |

### 三、创新性（权重 20%）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 情节设计 | 10分 | 是否有新意 |
| 人物塑造 | 10分 | 是否立体饱满 |

### 四、完整性（权重 20%）

| 细项 | 分值 | 评估要点 |
|------|------|----------|
| 内容完整 | 10分 | 结构是否完整 |
| 钩子设置 | 10分 | 悬念是否得当 |

---

## 问题分级

| 级别 | 标记 | 说明 | 后续动作 |
|------|------|------|----------|
| 严重 | 🔴 | 影响逻辑或世界观一致性 | 生成 🔴 高优先级 TODO |
| 中等 | 🟡 | 影响阅读体验 | 生成 🟡 中优先级 TODO |
| 轻微 | 🟢 | 可优化但不影响核心 | 生成 🟢 低优先级 TODO |

## 任务类型

### 完整评审（full）

```
输入：
- 章节内容
- 项目设定
- 历史评审（如有）

执行：
- 四维度完整评分
- 详细问题分析
- 改进建议

输出：
- 完整评审报告
- 归档路径：memory/projects/{project}/reviews/第{n}章-评审报告.md
```

### 快速评审（quick）

```
输入：
- 章节内容

执行：
- 核心维度评分（逻辑、可读性、完整性）
- 关键问题识别（最多3个）
- 核心建议（最多3个）

输出：
- 简化评审结果
```

---

## 完成后动作

评审完成后，XiaoYe 自动执行：

1. Write 评审报告到 `memory/projects/{project}/reviews/`
2. 根据问题级别，Edit `todo.md` 添加修改任务：
   ```
   - 🔴 修改第{n}章：{问题描述}
   - 🟡 优化第{n}章：{问题描述}
   - 🟢 润色：{问题描述}
   ```
3. Edit `progress.md` 更新评审分数和趋势
4. ⭐ **如章节被修改，[MODIFY_TRIGGER] 触发管理模块更新章节简介**

## 示例执行

```
用户：评审第二章

XiaoYe 执行：
[Read] novels-archive/{项目名}/章节/第二章-{标题}.md
[Read] rules/评审标准.md
[Read] novels-archive/{项目名}/设定/

[评分]
- 逻辑一致性：27/30（因果链条9，人物动机9，世界观9）
- 可读性：25/30（语言8，信息密度9，情感8）
- 创新性：17/20（情节9，人物8）
- 完整性：18/20（内容9，钩子9）
- 总分：87/100，⭐4.4

[问题识别]
- 🟡 第3段：战斗描写节奏稍快，可增加过渡
- 🟢 第7段：对话略显书面化，可口语化

[改进建议]
- 优先级1：战斗场景增加感官细节描写
- 优先级2：对话根据人物性格调整语气

[Write] memory/projects/{项目名}/reviews/第2章-评审报告.md

[Edit] memory/projects/{项目名}/todo.md
添加：- 🟡 优化第2章：战斗节奏过渡

[Edit] memory/projects/{项目名}/progress.md
更新：最新评审分=4.4，质量趋势=提升

⭐ 如章节被修改，[MODIFY_TRIGGER] 触发管理模块：
[Edit] novels-archive/{项目名}/章节/章节简介.md
更新第二章简介条目

[输出] 评审完成：总分87分⭐4.4，趋势↑，待办已更新
```

## 与创作模块的协作

在"创作并评审"流程中：

```
[Writer Module] 完成 → 归档 + ⭐更新章节简介
    ↓
[Reviewer Module] 评审 → 归档报告 → 更新TODO
    ↓
    └─ 如章节被修改 → ⭐[MODIFY_TRIGGER] 更新章节简介
    ↓
[输出] 创作摘要 + 评审摘要 + TODO更新
```
