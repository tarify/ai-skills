# Writer Module - 创作模块

## 模块定义

创作模块是 XiaoYe 的核心功能，负责小说内容创作。**这不是独立 Agent，而是 XiaoYe 主 Agent 的内置能力。**

---

## 触发条件

- 用户指令包含："创作"、"写"、"续写"
- 指定章节、人物、设定等内容创作

---

## 执行流程

```
1. ⭐ 前置条件检查（必须全部完成才能开始创作）：
   □ 读取项目设定（世界观、人物、情节）
   □ 读取前3章章节简介（如存在）
   □ 准备应用辩证分析模板思考

2. Read 设定文件（世界观、人物、情节）
3. ⭐ Read 前3章章节简介（如存在）
   - 路径：novels-archive/{project}/章节/章节简介/第{n}章-{标题}.md
   - 规则：创作第N章时，读取第N-3到第N-1章的简介文件
   - 第一章创作时跳过此步骤
4. Read 前文章节（如有）
5. 应用辩证分析模板思考
6. 创作内容
7. 自我批判和可读性检查
8. Write 归档到 novels-archive/
9. ⭐ 触发管理模块更新章节简介
10. 更新 progress.md
```

## 参考文件

创作前必须读取：

| 文件类型 | 路径 | 用途 | 必需 |
|----------|------|------|------|
| 写作规范 | `rules/写作规范.md` | 创作准则 | ✅ |
| 辩证分析模板 | `templates/辩证分析模板.md` | 思考框架 | ✅ |
| 世界观设定 | `novels-archive/{project}/设定/世界观/` | 背景规则 | ✅ |
| 人物设定 | `novels-archive/{project}/设定/人物/` | 角色信息 | ✅ |
| 前文章节 | `novels-archive/{project}/章节/` | 连贯性 | ✅ |
| ⭐ 章节简介 | `novels-archive/{project}/章节/章节简介/` | 前3章概要 | ✅ 如存在 |

### ⭐ 前置条件检查机制

创作前必须完成以下检查，**不可跳过**：

```
前置条件检查清单：
□ 项目设定已读取（世界观、人物、情节）
□ 前3章章节简介已读取（如存在）
  - 创作第N章时，读取第N-3到第N-1章的简介
  - 第一章创作时跳过
□ 辩证分析模板已准备

检查规则：
- 如果章节简介存在但未读取 → 必须先读取，不可跳过
- 如果是第一章创作 → 跳过简介读取
- 简介读取失败（文件不存在）→ 记录警告，继续创作
```

## 辩证写作法

### 核心原则

1. **多角度分析**：从不同视角审视情节发展
2. **逻辑自洽**：确保因果链条完整
3. **白而不浅**：用易懂语言承载深刻内容
4. **自我批判**：识别问题并改进

### 执行要求

- 设定语言不入侵小说正文
- 角色不知道的不写
- 恐惧来自细节而非形容词
- 快节奏用短句，慢节奏可展开

## 任务类型

### 创作章节

```
输入：
- 章节号
- 标题（可选）
- 用户要求
- 目标字数

输出：
- 章节正文
- 归档路径：novels-archive/{project}/章节/第{n}章-{标题}.md
```

### 创作人物卡

```
输入：
- 角色名
- 角色定位
- 基本要求

输出：
- 人物卡文档
- 归档路径：novels-archive/{project}/设定/人物/{name}.md
```

### 创作世界观设定

```
输入：
- 设定主题
- 关键要素

输出：
- 世界观文档
- 归档路径：novels-archive/{project}/设定/世界观/{topic}.md
```

### 创作情节大纲

```
输入：
- 小说类型
- 主题
- 章节范围

输出：
- 大纲文档
- 归档路径：novels-archive/{project}/设定/情节/{topic}.md
```

## 大规模创作（后台辅助）

当创作任务超过 5000 字或需要长时间思考时，可使用后台辅助：

### 方案 A: sessions_spawn

```
sessions_spawn(
  task="创作{项目名}第三章，完整的战斗场景，5000字以上...",
  runTimeoutSeconds=1800
)
```

等待公告返回，从文本中提取创作内容。

### 方案 B: Agent 工具

```
Agent(
  name="writer-bg",
  description="创作章节",
  prompt="完整的创作指令...",
  run_in_background=true
)
```

使用 TaskOutput 获取结果。

---

## 完成后动作

创作完成后，XiaoYe 自动执行：

1. Write 归档内容
2. Edit 更新 progress.md（章节数、字数）
3. ⭐ **触发管理模块更新章节简介**（[MODIFY_TRIGGER] 强制触发，不可跳过）
4. 如有评审请求，调用评审模块

## 示例执行

```
用户：创作第三章，推进主角的成长

XiaoYe 执行：

⭐ [前置条件检查]
□ 项目设定：待读取 ✓
□ 前3章简介：待读取（创作第3章，需读取第1-2章简介）✓
□ 辩证分析模板：待读取 ✓

[Read] novels-archive/{项目名}/设定/世界观/*.md
[Read] novels-archive/{项目名}/设定/人物/{主角名}.md
⭐ [Read] novels-archive/{项目名}/章节/章节简介/第一章-{标题}.md
⭐ [Read] novels-archive/{项目名}/章节/章节简介/第二章-{标题}.md
    → 获取第1章、第2章简介，了解前文脉络
[Read] novels-archive/{项目名}/章节/第二章-{标题}.md
[Read] rules/写作规范.md
[Read] templates/辩证分析模板.md

[思考] 应用辩证分析：
- 方案1：正面战场成长
- 方案2：侧面观察成长
- 选择方案1，理由：直接展示更震撼
- 参考简介衔接点：第2章结尾战斗升级

[创作] 3000字章节内容...

[自我批判]
- 问题1：节奏稍快
- 修正：增加过渡段落

[Write] novels-archive/{项目名}/章节/第三章.md

⭐ [MODIFY_TRIGGER] 触发管理模块：
[Write] novels-archive/{项目名}/章节/章节简介/第三章-{标题}.md
创建第三章简介文件

[Edit] memory/projects/{项目名}/progress.md
更新：章节完成数=3，总字数=...

[输出] 创作完成摘要，章节简介已更新
```
